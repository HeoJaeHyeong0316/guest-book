## 2. 기능 상세 정의 (Functional Requirements)

### 2.1 게시글 관리
- **글 작성:** 하단 입력창을 통해 짧은 메시지 작성 및 등록.
- **실시간 목록:** 작성된 글은 페이지 새로고침 없이 즉시 상단에 노출 (Supabase Realtime).
- **탭 구분:** '전체 글'과 '내 글'을 탭으로 구분하여 필터링 조회.
- **좋아요:** 게시글 우측 하단 하트 아이콘을 클릭하여 공감 표시.

### 2.2 사용자 경험 (UX)
- **프로필:** 가입 시 부여된 기본 아바타와 닉네임 표시.
- **날짜 표시:** `YYYY.MM.DD` 형식으로 작성 일자 표시.
- **알림:** 상단 종 아이콘을 통해 새로운 반응 확인 (UI만 구현).

---

## 3. UX Flow

1. **메인 진입:** "우리 반 방명록" 헤더와 함께 전체 글 목록 확인.
2. **탭 전환:** '전체 글'과 '내 글' 사이를 스와이프 또는 클릭으로 이동.
3. **글 작성:** 하단 입력창 클릭 -> 메시지 타이핑 -> '등록' 버튼 클릭.
4. **결과 확인:** 작성된 글이 최상단에 애니메이션과 함께 등장.
5. **인터랙션:** 마음에 드는 글에 '좋아요' 클릭.

---

## 4. 기술 스택 (Tech Stack)

- **Frontend:** Next.js (App Router), CSS Modules (Vanilla CSS 스타일링)
- **Backend/DB:** Supabase (Database, Realtime)
- **Version Control:** GitHub
- **Deployment:** Vercel

---

## 5. 디자인 가이드 (UI Design)

- **Main Color:** Peach/Beige (#FFF5EE 또는 유사한 파스텔톤)
- **Point Color:** Black (#000000), Red (#FF4D4D for Hearts)
- **Typography:** 현대적이고 가독성 높은 산세리프 폰트 (Pretendard 추천)
- **Shape:** 전체적으로 둥근 모서리 (Border-radius: 20px 이상) 및 부드러운 그림자 활용
